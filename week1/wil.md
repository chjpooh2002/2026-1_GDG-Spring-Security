<인증 / 인가>
-우리가 보안을 생각할 때는 크게 2가지 단계를 거치며 판단한다.
-처음으로 이 데이터가 DB에 있는지 검증하는 인증의 절차고 있고 인증 절차 후에 수행하는 것이 바로 인증이다.
-인가 절차는 DB에 특정 데이터가 접근할 수 있는 영역인지를 확인하는 단계로 인증된 데이터 중에서도 일부만이 들어갈 수 있다. 
-> 예를 들어 관리자 페이지는 승인된 몇 명의 관리자만 들어갈 수 있는 페이지이므로 인가 절차에서 관리자 외의 데이터의 접근을 막는다.

<서블릿 필터>
-Spring Security는 서블릿 필터라는 기술을 사용한다.
    ㄴ여기서 서블릿 필터란?  요청 / 응답을 가로채서 전처리/후처리 하는 장치 ex) 요청 로깅, 인코딩 설정
-http 요청이 들어오면 -> Filter chain을 지나면서 인증 -> 그 다음 인가 판단
- Filter에서 대부분이 처리되고 Controller까기 가기 전에 이미 "통과 가능 여부"가 결정됨

<DTO를 Record 타입으로 설계할 때의 이점>
- 생성자, getter, equals, hashCode / toString 전부 자동 생성되어 코드가 짧다.
- 불변 객체로 값이 한 번 생성되면 변하지 않는다. (Spring에서 DTO는 보통 요청/응답용 데이터 전달용이므로 중간에 값이 변경되면 버그가 발생할 확률이 높음)
- record는 선언 자체가 의도가 명확하여 협업 시 매우 좋다.
- DTO를 Record 타입으로 바꾸는 과정에서 record는 getter 이름이 getUsername()이 아니라 그냥 username(), password()이므로 코드를 수정해주었다.
